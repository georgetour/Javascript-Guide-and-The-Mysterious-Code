A Javascript Guide and The Mysterious Code.

A guide to javascript that will help you learn javascript and some mysterious code it hides. I will try to explain as simple as possible for beginners or even for those need a refreshing.

Prologue

What is Javascript?

Javascript is the programming language of the web. When we had the first internet pages, everything was simple with some text inside it and some very basic styling.

Since then, world wide web has evolved tremendously and users interact with pages in many ways. A pop up window, menus changing, image sliders and anything that you see in modern pages that increases their experience.

Here is where javascript comes. Thanks to javascript you can manipulate the page with many amazing ways.

Do you need javascript?

Yes, yes and yes. If you are serious about web developing and not only then javascript is a must.

HTML5 and CSS3 are really powerful, but for anything web related to shine you must have javascript.

When you have your first application or website that does something thanks to javascript, you will say WOW!

End prologue

4 ways to call it :

First not recommended to add these lines at head tag.

So we are saying to our HTML to use an external script since we have the src attribute at js/main.js which is of type text/javascript.

Reload the page to see that we will get our alert.

This way is not recommended, unless a script must start that manipulates CSS,images etc.Imagine that page starts to run the code from top to bottom. When it finds this script the page will stop to load this script and then continue. That' s bad user experience, search engines don't like your page to be slow and you will see that speed measurements tools penalize you for that like google pagespeed insights or gtmetrix.

Second way old school but correct and no browser problems.

Just put the external script code at the end of the body.

It will load the script when everything have finished so no problems with that. If you have many scripts just put them below and they will run from top to bottom.

Third way async attribute.

Just put the external script code at the head of the body with async attribute.

This is a good way that doesn't stop page loading. It loads the script asynchronous with the page. Though you might see some strange behaviors if you have many scripts that are depended each other.

Forth way defer attribute.

Just put the external script code at the head of the body with defer attribute.

This is a recommended way to load the script since it won't stop the page from loading. It loads the script at the end in order. So we will have after the page load main.js and then main2.js.
